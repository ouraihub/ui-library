# 架构决策记录（ADR）

本目录记录 @ouraihub/ui-library 项目的重要架构决策。

## 什么是 ADR？

架构决策记录（Architecture Decision Record）是一种记录重要架构决策的文档格式。每个 ADR 描述一个决策的背景、选择、理由和后果。

## 为什么需要 ADR？

- **记忆**: 记录当初为什么这样设计，避免遗忘
- **传承**: 帮助新成员理解设计背景
- **审查**: 便于回顾和质疑过去的决策
- **学习**: 从成功和失败中学习

## ADR 格式

每个 ADR 包含以下部分：

1. **标题**: 简短描述决策内容
2. **状态**: 提议中 | 已接受 | 已废弃 | 已替代
3. **背景**: 为什么需要做这个决策？
4. **决策**: 我们决定做什么？
5. **理由**: 为什么这样决策？
6. **后果**: 这个决策带来什么影响？
7. **替代方案**: 考虑过哪些其他方案？为什么没选？
