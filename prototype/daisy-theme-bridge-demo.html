Theme bridge demo · 不侵入 core / preset

把 daisyUI 主题能力封成你自己的组件，是合理的。

这个 demo 模拟了一个 `ThemeSwitcher + DaisyThemeBridge` 的最小产品形态。外部只面对 `light / dark / playful / nature` 这些你自己的语义主题，内部再映射到 daisyUI 主题名。

这个 demo 想证明什么

不是把 daisyUI 硬塞进核心层，而是把它变成一个可替换的主题提供者。以后即使换掉 daisyUI，外部 API 也不必变。

这块可以想象成 docs / dashboard / landing shell。你不需要让业务页面直接知道 daisyUI 的 `cupcake` 或 `emerald`，只需要暴露你自己的主题语义。
const bridge = new DaisyThemeBridge({ target: document.documentElement, storageKey: "ouraihub-theme", themes, defaultSemanticTheme: "light" })
