文档预设 · 导航优先 · 技术内容站点

它给人的感觉是：我马上能找到要看的章节，并且能直接复制代码。

这一套更像开发者文档中心。左边是导航树，中间是正文，右边是目录。页面强调代码块、提示框、搜索、面包屑和 API 参考区块。

Note: 适合产品文档、SDK 文档、组件库文档，而不是偏情绪表达和连续阅读的内容专栏。
Tip: 你会默认得到 Sidebar、TOC、CodeBlock、Callout、Breadcrumb、SearchModal、ApiReference 这些文档组件角色。

页面骨架是“查找效率”导向

和博客预设不一样，文档预设不会把“文章氛围感”放在前面。它先考虑的是导航深度、跳转效率、技术信息密度，以及用户能否快速定位到参数、步骤和示例代码。

import { docsPreset } from '@ouraihub/preset-docs'

export default {
  presets: [docsPreset]
}

典型内容块会长这样

除了正文，它还会天然适合承载 API 参数表、告警说明、代码块、搜索入口和分页导航。你可以把它想成一套给“说明型页面”准备好的版式和默认气质。

Field
Type
Description
presets
Preset[]
注册一个或多个预设，统一注入 token、组件配置和布局配置。
layout
LayoutConfig
控制文档页面骨架，比如 sidebar、content、toc 的区域关系。
components
ComponentConfig[]
定义 CodeBlock、Callout、SearchModal、Breadcrumb 等默认行为。
Warning: 如果你的主要目标是展示观点、故事和系列文章，而不是技术信息检索，这套会显得太“工具化”。
