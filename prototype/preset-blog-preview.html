文章详情页会更像这样

文章内容居中，文字更大，阅读节奏更慢，适合长文。

作者 OurAI Hub · 2026-05-19 · 分类：架构 / 设计系统 / 内容策略

这类预设默认会让正文区域更像“阅读器”。主内容通常限制宽度，减少一行过长带来的疲劳，同时把行高放宽，让眼睛移动更轻松。

目录、作者简介、评论区和相关文章并不是主角，它们是围绕正文服务的。页面核心是“把一篇文章读完”，不是“快速跳转到另一个 API 页”。

直观理解：如果你的网站希望用户停留、阅读、感受观点，博客预设是顺手的；如果用户主要是查参数、复制代码、顺着导航找答案，它就不够对味。
