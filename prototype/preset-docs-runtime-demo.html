Documentation preset

这是更接近真实落地的文档预设页面，而不是概念演示页。

它直接把 preset 里真实存在的 `Sidebar`、`TableOfContents`、`CodeBlock`、`Callout`、`SearchModal`、`ApiReference` 和 docs layout 翻译成一个接近产品文档站的运行页面。

Note: 文档预设首先关心的是导航效率、信息分层、代码展示和查阅路径，而不是文章氛围。
Tip: 你能明显看到它天然适合三栏结构：左边导航、中间正文、右边目录。

它更像“说明书”和“参考中心”

这套预设的真实能力重心不在于讲一个故事，而在于帮助用户快速找到一个章节、定位一个 API、复制一段代码、理解一个配置项。也因此，`Sidebar`、`TableOfContents`、`Breadcrumb`、`SearchModal` 会自然成为主角。

页面里的代码块、提示框、分页和 API 表格，都不是额外加戏，而是这类站点最常见的核心内容块。

import { docsPreset } from '@ouraihub/preset-docs'

export default {
  presets: [docsPreset]
}

Callout 和 CodeBlock 会很自然

文档预设内置的语义色就为 `note / tip / warning / danger` 准备好了背景、边框和文字色。代码块也单独定义了深色背景、等宽字体、复制按钮这一类典型文档体验。

Warning: 如果你拿它去做内容专栏，会明显感觉它太“结构化”、太“工具型”。
Danger: 如果用户预期是连续阅读文章，而你给的是这种文档骨架，体验会偏硬。

API Reference 区块会像这样

Field
Type
Description
layout
LayoutConfig
定义 sidebar、content、toc 等区域的尺寸和展示方式。
components
ComponentConfig[]
定义 CodeBlock、Callout、ApiReference、SearchModal 等默认行为。
tokens
DesignTokens
控制代码高亮、侧边栏、提示框、排版、间距和颜色系统。

所以它给人的真实感受是什么

打开这种页面，用户首先会做的是导航、搜索、定位、复制，而不是沉浸阅读。它不是在把文章讲得更好，而是在把信息组织得更清楚。
